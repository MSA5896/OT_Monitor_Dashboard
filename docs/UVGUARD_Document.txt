1. PROJECT OVERVIEW & SYSTEM ARCHITECTURE
1.1 Project Name & Objective
UV-GUARD is an autonomous, safety-interlocked UV-C disinfection robot integrated with a real-time environment monitoring station, purpose-built for Operating Theatres (OT) and Intensive Care Units (ICU). The system ensures zero human exposure during UV disinfection, complete spatial coverage through AI-guided navigation, and continuous post-disinfection environment quality reporting aligned with WHO, NABH, and ASHRAE standards.

2. SYSTEM SAFETY LOGIC & INTERLOCK PROTOCOL
2.1 Safety State Machine (FSM)
UV-GUARD operates a strict 7-state Finite State Machine. UV-C lamps CANNOT energize unless ALL safety conditions are simultaneously TRUE. A single breach triggers immediate lamp shutdown.
State
Name
Condition to Enter
Action
S0
IDLE
System power-on default
All outputs OFF. Await start command.
S1
PRE-CHECK
Start command received
Run thermal scan + check all door/window sensors.
S2
ARMED
No humans detected + all doors/windows CLOSED
Alert via buzzer/light. 30-sec countdown before UV ON.
S3
DISINFECTING
All S2 conditions still TRUE after countdown
UV-C ON. Robot begins coverage pattern. Continuous monitoring.
S4
BREACH DETECTED
Any sensor fails during S3
UV-C OFF immediately (<50ms). Sound alarm. Log event.
S5
RE-CHECK
Breach cleared + operator confirmation
Return to S1 pre-check cycle.
S6
COMPLETE
Coverage 
map
 100% + timer elapsed
UV-C OFF. Post-scan mode begins. Generate report.
2.2 Sensor Interlock Conditions
Thermal Human Detection
Sensor: MLX90640 32×24 Thermal Camera Array (×2 units for full room coverage)
Detection threshold: Any pixel >32°C within defined ROI = HUMAN DETECTED
Scan frequency: Every 500ms during ARMED and DISINFECTING states
False-positive handling: 3 consecutive positive readings required before breach trigger (1.5 sec)
Door & Window Sensors
Type: Magnetic Reed Switches (MC-38) on all OT/ICU doors + windows
Logic: NC (Normally Closed) = SAFE. Any OPEN = BREACH
Response time: <10ms interrupt-driven on microcontroller GPIO
Wiring: Supervised loop with EOL (End of Line) resistor to detect tampering
UV-C Interlock Hardware
Hardware relay with normally-open contacts – relay de-energizes = UV OFF (fail-safe)
Separate hardware watchdog timer: if MCU freezes → relay trips within 2 seconds
Emergency Stop button (red mushroom head) on robot + wall-mounted panel
2.3 Safety Wiring Diagram (Description)
Power → Main MCB → Safety Relay Chain: [Thermal OK] AND [All Doors Closed] AND [Watchdog Pulse] AND [No E-STOP] → UV Lamp Driver Board. Any link breaks = UV OFF.

4.5 UV Fluorescence Imaging System (Organism Detection)
Concept
Under UV-A (365nm) illumination in a darkened room (blackout condition after UV-C cycle), biological materials fluoresce. Blood stains fluoresce red-pink, bacteria colonies show as blue-white, fungi show green-yellow, and tissue particles show orange. This allows post-disinfection verification.
Hardware for UV Fluorescence Scanning
UV-A LED Panel: 365nm UV-A (blacklight) – 20W panel mounted on robot arm
Camera: Raspberry Pi HQ Camera Module with 16mm lens (no IR filter – sensitive to fluorescence)
Optical Filter: 400nm long-pass optical filter on camera lens (blocks UV, passes visible fluorescence)
Room condition: OT blackout blinds + door closed (pre-programmed blackout routine)
Detection Pipeline
Capture high-res image under UV-A in darkened room
Color
 segmentation: HSV thresholding for red (blood), 
green-yellow
 (fungi), 
blue-white
 (bacteria)
CNN classifier (MobileNetV2 fine-tuned) for 4 classes: Clean, Blood, Bacteria/Fungi, Tissue
Overlay detection map on floor plan and display on HMI with GPS-tagged location
Risk score: % contaminated area vs total floor area
